# 作業実績書: OCRバッチ整備

## 概要
`yomitoku` を使った D&D 実行向け OCR バッチを作成し、出力先の運用（`figure` / `analysis_results`）と仮想環境有効化を自動化した。あわせて README と `.gitignore` を整備し、利用手順と成果物管理を明確化した。

## ゴール
- D&D で PDF / 画像 / 画像フォルダを OCR できること
- 図抽出は `figure`、詳細解析結果は `analysis_results` に出力されること
- 実行前に仮想環境を確実に有効化すること
- README と `.gitignore` が実運用に耐える状態であること

## 手順
1. 既存ファイル（`README.md`、`.gitignore`、`main.py`）を確認した。
2. `drag_and_drop_ocr.bat` を新規作成し、以下を実装した。
   - `.venv\Scripts\activate.bat` の呼び出し
   - `yomitoku` コマンド存在確認
   - 入力判定（PDF、画像、フォルダ）
   - 出力先固定（`analysis_results`、`figure`）
   - 複数入力対応、未対応拡張子のスキップ
3. `README.md` に仕様、セットアップ、使い方、注意点、参考情報を追記した。
4. `.gitignore` に OCR 出力ディレクトリと Python 生成物を追記した。
5. `cmd /c drag_and_drop_ocr.bat` で引数なし起動時メッセージを確認した。
